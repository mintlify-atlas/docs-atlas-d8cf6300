---
title: "Target options"
description: "Flags for specifying which hosts RelayKing scans: individual IPs, CIDR ranges, files, Active Directory audit mode, and session resume."
---

At least one of the following must be provided: a positional `target`, `--target-file`, `--audit`, `--coerce-all`, or `--session-resume`.

<Tip>
  A `targets.txt.example` file is included in the repository showing the supported line formats.
</Tip>

## Flags

<ParamField path="target" type="string[]">
  One or more positional target arguments. Accepted formats:

  - Single IP: `10.0.0.5`
  - Hostname or FQDN: `dc01.corp.example.local`
  - CIDR notation: `10.0.0.0/24`
  - Hyphenated IP range: `10.0.0.1-10.0.0.50`

  Multiple values can be specified. When supplying a single target, place it at the end of the argument list.
</ParamField>

<ParamField path="-t / --target-file" type="string">
  Path to a file containing targets, one per line. Supports the same formats as the positional `target` argument (IPs, hostnames, CIDRs, ranges). Blank lines and lines beginning with `#` are ignored.
</ParamField>

<ParamField path="--audit" type="boolean" default="false">
  Audit mode: enumerate all computer accounts from Active Directory via LDAP and scan them. Requires low-privilege AD credentials (`-u`, `-p`, `-d`) and a reachable DC (`--dc-ip`). Default protocols when `--audit` is active are `smb`, `ldap`, `ldaps`, and `mssql`; adding `http,https` enables tier-0 HTTP relay path analysis.

  <Note>`--audit` cannot be combined with `--coerce-all`.</Note>
</ParamField>

<ParamField path="--no-ping" type="boolean" default="false">
  Skip the ICMP ping sweep that RelayKing uses to filter live hosts from CIDR ranges before scanning. Use this when running through a SOCKS proxy, where ICMP is typically unsupported.
</ParamField>

<ParamField path="--session-resume" type="string">
  Path to a `.resume` session file generated by a previous `--audit` scan. Resuming skips AD enumeration, DNS resolution, and port scanning for work that was already completed, and continues from where the interrupted scan left off.

  The file must exist; RelayKing exits with an error if the path is not found.
</ParamField>

## Targeting modes

<Tabs>
  <Tab title="Single host">
    Scan a single IP address or hostname:

    ```bash
    python3 relayking.py -u lowpriv -p 'Summer2024!' -d corp.example.local \
      --dc-ip 10.0.0.1 --protocols smb,ldap 10.0.0.5
    ```

    ```bash
    python3 relayking.py -u lowpriv -p 'Summer2024!' -d corp.example.local \
      --dc-ip 10.0.0.1 --protocols smb,ldap dc01.corp.example.local
    ```
  </Tab>
  <Tab title="CIDR range">
    Scan an entire subnet. RelayKing pings hosts first to find live targets:

    ```bash
    python3 relayking.py -u lowpriv -p 'Summer2024!' -d corp.example.local \
      --dc-ip 10.0.0.1 --protocols smb,ldap 10.0.0.0/24
    ```

    Skip the ping sweep when scanning through a SOCKS proxy:

    ```bash
    proxychains python3 relayking.py -u lowpriv -p 'Summer2024!' -d corp.example.local \
      --dc-ip 10.0.0.1 --no-ping --protocols smb,ldap 10.0.0.0/24
    ```
  </Tab>
  <Tab title="IP range">
    Scan a hyphenated range:

    ```bash
    python3 relayking.py -u lowpriv -p 'Summer2024!' -d corp.example.local \
      --dc-ip 10.0.0.1 --protocols smb,ldap 10.0.0.1-10.0.0.50
    ```
  </Tab>
  <Tab title="Target file">
    Load targets from a file (IPs, hostnames, CIDRs, or ranges — one per line):

    ```bash
    python3 relayking.py -u lowpriv -p 'Summer2024!' -d corp.example.local \
      --dc-ip 10.0.0.1 --protocols smb,ldap -t targets.txt
    ```

    Example `targets.txt`:

    ```text
    10.0.0.5
    10.0.0.10-10.0.0.20
    10.0.1.0/24
    dc01.corp.example.local
    ```
  </Tab>
  <Tab title="Audit mode">
    Enumerate all AD computers and scan them automatically:

    ```bash
    python3 relayking.py -u lowpriv -p 'Summer2024!' -d corp.example.local \
      --dc-ip 10.0.0.1 --audit --protocols smb,ldap,ldaps,mssql,http,https \
      --proto-portscan --threads 10 -vv -o plaintext,json --output-file relayking-scan
    ```
  </Tab>
  <Tab title="Session resume">
    Resume an interrupted audit scan:

    ```bash
    python3 relayking.py -u lowpriv -p 'Summer2024!' -d corp.example.local \
      --dc-ip 10.0.0.1 --session-resume relayking-20240601.resume
    ```
  </Tab>
</Tabs>
